About the project

The goal of this project is to define the way to use a browser database. It exists a lot of javascript libraries to implement a browser database, but we just will tests to most used or followed.

Technologies

All libraries (tested), have the possibility to store (adapter) the database as an IndexedDB or WebSQL. The Browser compatibility will be directly linked with the choice of the adapter:

  • Indexeddb : Is implemented yet for all populare browser. But some functionnalities are not compatible with IE (caniuse)
  • WebSQL : Is not implement for Firefox and IE (caniuse)

Pouchdb, Forerunnerdb and lokijs are using by default the Indexeddb as default adapter.

Tests

For the test of the javascript database, we perform the following actions:

  • loading simple data
  • saving the data into an indexeddb
  • displaying the data directly on the browser

Results

  • For the manipulation of the database, all libraries have the same functionnalities.
  • Forerunnerdb has modules (data binding and templating) for the displaying of the data like table, grid or graphs. But this library need other external library (jQuery, etc...; depending of the used module)
  • About the documentation, Pouchdb is the really well documented. The others libraries have only a minimalist documentation.
  • Pouchdb is asynchronous
  • Performances: Better performance with PouchDB and Lokijs for querying/displaying data

Comparison

Why choosing Forerunner?

  • Modules for displaying data in templates
  • Data binding
  • Displaying graphs, grid and table very easily

Why choosing Lokijs?

  • View functionnalities for displaying complex request

Why choosing Pouchdb?

  • 4'571 stars on Github -> Big community
  • Wiki with a lot of How-to articles
  • Is asynchronous
  • Well documented
  • Bulk operations
  • A lot of tools for the development
  • Synchronization between local and remote database easier
